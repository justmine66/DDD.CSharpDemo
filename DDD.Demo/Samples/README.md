# 通用语言
玩家（Player）可以是战士（Fighter）、法师（Mage）、龙骑（Dragoon）
怪物（Monster）可以是兽人（Orc）、精灵（Elf）、龙（Dragon），怪物有血量
武器（Weapon）可以是剑（Sword）、法杖（Staff），武器有攻击力

魔法攻击：法师发起的攻击。

# 业务规则
1、玩家可以装备一个武器。
2、武器攻击可以是物理类型（0），火（1），冰（2）等，武器类型决定伤害类型。
3、兽人对物理攻击伤害减半。
4、精灵对魔法攻击伤害减半。
5、龙对物理和魔法攻击免疫，
6、龙骑对龙伤害加倍。

# 需求变更1
战士只能装备剑
法师只能装备法杖

# 需求变更2
战士和法师都能装备匕首（dagger）

# 需求变更终极
支持增加任意一种类型的玩家、怪物、武器、规则，都不违背 OCP(Open-Closed Principle) 原则。

一、比如：
 1、增加一种武器类型：狙击枪
 2、增加一条业务规则：狙击枪无视所有防御一击必杀

实现：
1、新增武器子类
2、修改玩家装备逻辑
3、修改怪兽伤害计算逻辑

二、比如：
1、增加一条业务规则：玩家和怪兽需要具备移动、跳跃、跑步能力。

实现：
   玩家和怪兽分别重复实现？
   封装能力系统，玩家和怪兽依赖来实现。
